# Keepers

Maintainers, contributors, and governance.

## Project Keeper

**Primary:** dhaupin (GitHub founder)

## How to Contribute

1. Fork the repo
2. Create a branch
3. Make changes
4. Open a PR
5. Pass QC checks
6. Merge after review

## Roles

- **Maintainer** - Can merge PRs, manage releases
- **Contributor** - Submit PRs, report issues
- **User** - Use, star, share

## Governance

- Open source, MIT license
- BC-breaking needs major version
- No corporate control
- Community-driven
