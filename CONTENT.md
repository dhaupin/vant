# Vant Content Guide

This file defines the voice, tone, and style for all Vant project content.

---

## Our Tone

We write like engineers who also happen to be helpful. Clear. Direct. Slightly informal when appropriate, serious when it matters.

We are transparent that AI agents contribute to this project. We don't hide it, and we don't over-explain it.

---

## Principles

### 1. Use Short Dashes

Always use `-` instead of `—` or `–`.

- Right: "Git is memory - every commit is a checkpoint"
- Wrong: "Git is memory — every commit is a checkpoint"

### 2. Avoid AI Clichés

These phrases feel overused and inauthentic:

- "delve into"
- "leverage"
- "unlock"
- "seamless"
- "empower"
- "journey"
- "realm"
- "game-changer"
- "cutting-edge"
- "revolutionary"
- "innovative" (as a standalone adjective)
- "we're excited to" (empty enthusiasm)
- "let's dive in"
- "ultimate guide to"
- "ever wondered"

Instead, just say what you mean.

### 3. Be Specific, Not Abstract

- Instead of: "This feature empowers users to do more"
- Write: "This feature lets you transfer your brain between instances"

### 4. State Facts, Not Hype

- Don't: "Vant is a revolutionary system that will change AI forever"
- Do: "Vant persists AI memory across sessions via Git"

### 5. Active Voice

Subject-verb-object. Prefer present tense.

- "The CLI loads the brain" not "The brain is loaded by the CLI"

### 6. Code Is Documentation

Show, don't just tell. Include working commands. Link to real files.

### 7. Humans and AIs Both Read This

Write for both. A human should feel respected. An AI should get clear instructions.

---

## Formatting

- Headers: Title case for titles, sentence case for subheads
- Lists: Use `-` for bullets, not `*`
- Code: Inline `code` for commands, blocks for examples
- Links: Descriptive, not "click here"

---

## What We Focus On

Content should center on:

- How to use Vant (clear steps)
- What problems Vant solves (specific)
- Technical details (accurate)
- Tradeoffs (honest)

Avoid:

- Vague value propositions
- Over-promising
- Filling space with buzzwords

---

## Example

### Before (Cliché)

"Discover the future of AI memory with Vant - the ultimate solution to unlock seamless persistent intelligence across your agentic workflows."

### After (Our Style)

"Vant stores your AI's memory in Git. Each session inherits the previous one. Use branches to experiment with different brain configs."

---

## Applying This Guide

Run a pass on all content files when:

- Adding new docs
- Writing release notes
- Updating README
- Creating issues or discussions

Check for:
- Em dashes replaced with hyphens
- Cliché phrases removed
- Specific over abstract
- Active voice

---

*This guide applies to all text in the Vant project.*
